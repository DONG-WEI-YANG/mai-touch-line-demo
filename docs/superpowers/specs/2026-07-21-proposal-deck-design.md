## 設計決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| 交付形式 | 新的獨立 deck | 與應用場景/建商價值/硬體清單並列成提案組合 |
| 受眾 | 建商/建案業主 | 商業說服力優先，技術點到為止 |
| 分階段軸 | 時間軸導入階段（非規模分級） | 與既有硬體 deck 的三級部署（按規模）互補不重複 |
| 預算 | 不放金額 | 避免 deck 外流被當報價單 |
| 管線 | 沿用 hardware.build.cjs 的調色盤/圖示/卡片元件 | 三套 deck 視覺一致、同一維護管線 |

## 內容原則

- 整合狀態沿用硬體 deck 的 LIVE（已整合上線）/ CORE（建議必備）/ OPT（選配待接），
  依實際程式現況標註，不誇大（語音預約=平台自建麥克風管道；Google 音箱僅裝置控制）。
- 每個 Phase 頁：左硬體（含規格）、右解鎖功能 — 讓建商看懂「裝這些換到什麼」。
- Phase 對齊工程時點：P1 工程期/交屋前（機房弱電預留）、P2 交屋入住期、P3 營運成熟期。

## Phase × 硬體 × 功能對照

- **P0（開發優先）**：品牌化客製（名稱/主視覺/公設清單）、功能盤點與客製開發、
  雲端部署（Render/Vercel/LINE OA 申請）、接待中心示範環境（平板 ×1 + Nest 音箱 ×1，
  DRY_RUN 示範閘道）→ 價值：預售即展示、銷售現場加值、住戶資料預建
- **P1**：Render/Vercel/LINE/Twilio（LIVE）+ 路由防火牆 VLAN + PoE 交換器 +
  邊緣閘道主機 + UPS + 櫃檯平板 + Web Kiosk → 啟用：預約/公告/帳務/工單/包裹/
  停車（人工）/LINE 通知/語音代訂/AI 管家
- **P2**：大廳 QR 掃碼門禁 + 住戶 IoT（光/空調/窗簾/安防）+ Nest 音箱/顯示器 +
  Nest Wifi Pro → 解鎖：QR 通行、語音裝置控制、App 家居控制、訪客通行、IoT 排程
- **P3**：車牌辨識柵欄 + 智能包裹櫃 + 電梯介接 + Nest Hub Max + 自架 NLP +
  PostgreSQL + 雙 UPS 備援 → 解鎖：車牌放行、取件通知、電梯禮賓、公設看板、資料自主
